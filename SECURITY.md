# Security Policy

## Reporting a Vulnerability

Report vulnerabilities or security-sensitive specification issues privately to:

```text
security@aep.foundation
```

Alternatively, contact a repository owner directly through GitHub.

## What to Report

Please report issues that could affect:

- Agent or Service authentication.
- Replay protection.
- Credential issuance, presentation, or revocation.
- Identity enumeration or probing resistance.
- Privacy leakage from protocol fields or examples.
- Misleading security requirements in draft text.

## Public Discussion

Do not open a public issue for a vulnerability that could enable active
exploitation or identity probing before maintainers have had a chance to
review it.

Security fixes may be handled through private coordination first, followed by a
public pull request or advisory when appropriate.
